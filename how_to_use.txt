состав программы - 

vs_circuit_solver.py 
vs_solver_settings.py
vs_solver_test.py 
всего три модуля.


основные термины - 

1) ЦЕЛЕВАЯ МОДЕЛЬ - электронная схема, основная характеристика - 
	ВАХ - numpy массивы для напряжения и тока соответственно. 
	Может быть получена в результате - 
		а) прямого измерения в железе - тогда загружается функцией init_target_from_csvFile().
		б) spice моделирования - тогда используем функцию init_target_by_circuitFile() с spice *.cir 
			файлом, его содержащим

2) ШАБЛОН СХЕМЫ, ФАЙЛ ШАБЛОНА СХЕМЫ - файл описания схемы формата spice *.cir, ВАХ которого предположительно 
	близка к ВАХ целевой модели. Неизвестные номиналы радиокомпонентов заменены на символы замены '{}', 
	расширение файла *.cir_t. Устанавливается в set_circuit_template()

3) ТЕКУЩАЯ СХЕМА - электронная схема, полученная из шаблона схемы подстановкой конкретных значений 
	номиналов радиокомпонентов. Создается вызовом generate_circuitFile_by_values().
	Её имя в переменной circuit_SessionFileName 

4) СРАВНЕНИЕ ВАХ, функция СРАВНЕНИЯ ВАХ - мера того, насколько ВАХ текущей схемы приблизился к 
	ВАХ целевой модели. реализацию см. analysis_misfit()

5) МАСКА ОПТИМИЗИРУЕМЫХ ПАРАМЕТРОВ - список булевых значений, определяющий конкретный порядок 
	подстановки и оптимизации номиналов радиокомпонентов. Подробности см. в коде vs_circuit_solver.py



принцип работы - нам необходимо подобрать такие номиналы радиокомпонентов для схемы, чтобы 
в результате его моделирования ВАХ его максимально приблизился к ВАХ целевой модели. Используются функции 
математической оптимизации из пакета scipy.optimize, функция сравнения ВАХ - оптимизируемая, её аргументы - 
искомые номиналы. см. run_fitter(), fitter_subroutine()

описание работы по шагам -
1. Устанавливаем целевую модель - init_target_from_csvFile() либо init_target_by_circuitFile()
2. Устанавливаем шаблон схемы - set_circuit_template()
3. Устанавливаем маску оптимизируемых параметров исходя из модели.
4. запускаем run_fitter()
5. Сравниваем ВАХ в результате оптимизации.
	Если счастья нет, повторяем пп2-5 для других файлов шаблона схемы.
